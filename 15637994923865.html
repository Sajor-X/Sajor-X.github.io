Linux Top Tips

2019/07/22

Linux相关 

安装Python36

yum -y install sudo
sudo yum -y install epel-release
sudo yum -y update
sudo yum -y install python36
python3.6 -m venv py36env
source py36env/bin/activate

缺少python.h

yum search python3-devel
sudo yum install -y python3-devel.x86_64

监控

pip install memory_profiler
pip install psutil

安装mysql

sudo yum -y install wget
sudo wget -i -c http://dev.mysql.com/get/mysql57-community-release-el7-10.noarch.rpm
sudo yum -y install mysql57-community-release-el7-10.noarch.rpm
sudo yum -y install mysql-community-server

sudo systemctl start mysqld.service
sudo systemctl status mysqld.service
grep "password" /var/log/mysqld.log
mysql -uroot -p

set global validate_password_policy=0;

set global validate_password_length=1;

ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';

flush privileges;

创建新用户

GRANT USAGE ON suntwin.* TO 'suntwin'@'localhost' IDENTIFIED BY'123456' WITH GRANT OPTION;
username 替换为用户名
password 替换为密码

授权

grant all privileges on suntwin.* to suntwin@localhost identified by '123456';
flush privileges;

开启远程

GRANT ALL PRIVILEGES ON . TO 'root'@'%' IDENTIFIED BY '123456' WITH GRANT OPTION;

mysql创建数据库

CREATE DATABASE test2 DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;

django 数据库初始化

python manage.py makemigrations
python manage.py migrate

修改了model出现错误

先 python manage.py makemigrations --empty yourappname 生成一个空的initial.py
再 python manage.py makemigrations 生成原先的model对应的migration file

nginx安装

sudo yum -y install epel-release
sudo yum -y install nginx
sudo systemctl enable nginx
sudo systemctl start nginx

开启防火墙

sudo firewall-cmd --permanent --zone=public --add-service=http
sudo firewall-cmd --permanent --zone=public --add-service=https
sudo firewall-cmd --reload

centos7 开启8080端口

sudo firewall-cmd --reload

Ubuntu配置mongodb远程

1.修改配置文件mongodb.conf
命令：vim /etc/mongodb.conf
2.重启mongodb服务
命令：/etc/init.d/mongodb restart
3.防火墙开放27017端口
命令：iptables -A INPUT -p tcp -m state --state NEW -m tcp --dport 27017 -j ACCEPT
